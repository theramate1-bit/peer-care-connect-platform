# Glossary (customer app)

Plain-language definitions. **New to this area?** Read [`00-JUNIOR_DEV_START_HERE.md`](00-JUNIOR_DEV_START_HERE.md) first.

| Term                          | Definition                                                                                                                                                                                                                                                                                         |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Expo**                      | Toolkit for React Native; `theramate-ios-client` uses it — one codebase for iOS and Android.                                                                                                                                                                                                       |
| **EAS**                       | Expo Application Services — cloud builds (`eas build`) and optional updates (`eas update`).                                                                                                                                                                                                        |
| **Anon key**                  | Supabase **public** key (safe in app). It only does what **RLS** allows — not an admin password.                                                                                                                                                                                                   |
| **Deep link**                 | A URL (`https://...` or `myapp://...`) that opens your app on a specific screen.                                                                                                                                                                                                                   |
| **Universal Link / App Link** | OS feature: `https` links open the installed app instead of the browser (when configured).                                                                                                                                                                                                         |
| **Customer app**              | Client + guest experiences; documented under `docs/customer-app/`.                                                                                                                                                                                                                                 |
| **Practitioner app**          | Therapist-facing dashboards and practice management — **not** in this folder.                                                                                                                                                                                                                      |
| **peer-care-connect**         | Main web app (React + Vite) in the monorepo.                                                                                                                                                                                                                                                       |
| **theramate-ios-client**      | Expo native app; shared for iOS/Android; customer shell today.                                                                                                                                                                                                                                     |
| **Direct booking**            | `/book/:slug` — practitioner-specific booking link.                                                                                                                                                                                                                                                |
| **Mobile request**            | Requesting a **mobile** (traveling) therapist; uses `mobile_booking_requests`.                                                                                                                                                                                                                     |
| **Hybrid practitioner**       | Offers both clinic and mobile; `HybridBookingChooser` on web.                                                                                                                                                                                                                                      |
| **Marketplace**               | `/marketplace` — discovery surface for practitioners.                                                                                                                                                                                                                                              |
| **RLS**                       | Row Level Security — Postgres policies; same for web and native.                                                                                                                                                                                                                                   |
| **Edge Function**             | Deno functions on Supabase; privileged server logic.                                                                                                                                                                                                                                               |
| **Parity**                    | Native behavior matches web + database rules, not necessarily identical UI.                                                                                                                                                                                                                        |
| **Platform readiness**        | Backend/contracts vs UI vs payments vs links — see [`15-MOBILE_PLATFORM_READINESS.md`](15-MOBILE_PLATFORM_READINESS.md).                                                                                                                                                                           |
| **BMAD UX**                   | BMM workflow `create-ux-design` — formal UX spec; foundations in [`18-MOBILE_UI_UX_FOUNDATIONS_BMAD.md`](18-MOBILE_UI_UX_FOUNDATIONS_BMAD.md); stub in [`../../_bmad-output/planning-artifacts/ux-design-customer-mobile.md`](../../_bmad-output/planning-artifacts/ux-design-customer-mobile.md). |
| **PaymentSheet**              | Stripe React Native flow for card payment UI; still driven by server-created PaymentIntents / Checkout.                                                                                                                                                                                            |
| **SCR-\*** IDs                | Stable screen identifiers for design/QA — [`25-SCREEN_REGISTRY_FOR_DESIGN.md`](25-SCREEN_REGISTRY_FOR_DESIGN.md).                                                                                                                                                                                  |
| **Text wireframe**            | ASCII layout blocks in [`24-SCREEN_WIREFRAMES_AND_LAYOUTS.md`](24-SCREEN_WIREFRAMES_AND_LAYOUTS.md) until Figma replaces them.                                                                                                                                                                     |
