## How to use this table

- **Before changing behavior**: Check the row for that capability; use "Source of truth" to find code.
- **When fixing a bug**: Set "Consistent?" to No/Partial and add "Notes / Gap"; add a backlog item in [GUEST_VS_CLIENT_AUDIT_BACKLOG.md](GUEST_VS_CLIENT_AUDIT_BACKLOG.md).
- **When adding a feature**: Add a new row and fill Guest vs Client behavior so both paths stay consistent.

**Per–user-type docs (touchpoints for guests and clients):**

- **Guest:** [USER_TYPE_GUEST.md](USER_TYPE_GUEST.md) – Marketplace, GuestBookingFlow, MobileBookingRequestFlow (guest path), view by token, email/messaging, pre-assessment, account conversion.
- **Client:** [USER_TYPE_CLIENT.md](USER_TYPE_CLIENT.md) – Marketplace, ClientBooking page, BookingFlow, MobileBookingRequestFlow (client path), My Sessions, in-app messaging, reschedule/cancel.

See also: [GUEST_VS_CLIENT_AUDIT_BACKLOG.md](GUEST_VS_CLIENT_AUDIT_BACKLOG.md), [GUEST_VS_CLIENT_RULES.md](../development/GUEST_VS_CLIENT_RULES.md).
