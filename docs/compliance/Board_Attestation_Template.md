## Attestation

We, the directors of Theramate Limited, hereby attest that:

### 1. Governance & Accountability

✅ **Purpose defined and documented** - Location and IP tracking support legitimate, specific business objectives (marketplace matching, security, analytics)

✅ **Lawful basis validated** - Consent obtained for location tracking (PECR requirement); legitimate interests and consent for IP tracking

✅ **DPIA completed and approved** - Data Protection Impact Assessment completed for location and IP tracking, risks identified and mitigated

✅ **DPO review completed** - Data Protection Officer has reviewed compliance posture and risk

✅ **ROPA updated** - Records of Processing Activities updated with location and IP tracking activities, including data categories, flows, retention, and recipients

### 2. Consent, Transparency & User Control

✅ **Consent mechanism assessed** - Explicit, opt-in consent mechanism implemented for location tracking; compliant cookie/PECR controls for IP-based tracking

✅ **Withdrawal process verified** - Consent can be withdrawn easily and takes effect immediately

✅ **Privacy notice reviewed** - Privacy notice clearly explains what is collected, why, retention, rights, and third-party access

✅ **User choice respected** - Opt-outs and objections are technically enforced

### 3. Data Minimisation & Purpose Limitation

✅ **Data scope validated** - Only the minimum location/IP data required is collected

✅ **Secondary use prohibited** - Any new purpose requires a fresh lawful basis and transparency update

✅ **Anonymisation/pseudonymisation applied** - Identifiers reduced wherever possible (IP addresses anonymized after retention)

### 4. Security & Technical Controls

✅ **Access controls implemented** - Role-based access; privileged access monitored

✅ **Encryption applied** - In transit (TLS 1.3) and at rest (AES-256), especially for precise location data

✅ **Audit logging enabled** - Access to tracking data is logged and reviewed

✅ **Processor due-diligence completed** - Contracts include Article 28 requirements and PECR obligations

### 5. Retention & Deletion

✅ **Retention schedule approved** - Data kept only as long as necessary for the stated purpose

✅ **Automated deletion configured** - Technical enforcement of retention limits

✅ **Erasure process tested** - Requests handled within statutory timelines

### 6. Individual Rights & Redress

✅ **Rights-handling processes validated** - Access, rectification, objection, portability, and erasure workflows tested

✅ **Objection to tracking respected** - Legitimate interests assessments include a balancing test and opt-out route

✅ **Children's data safeguards assessed** - Age-appropriate design and parental consent where required (services currently 18+ only)

### 7. PECR Compliance (Location Data)

✅ **Location data classification confirmed** - Network-derived location data treated as PECR-regulated

✅ **Consent captured before processing** - No location tracking until explicit consent is obtained

✅ **Value-added service justification documented** - Service cannot function without location data and users understand this

### 8. Oversight, Reporting & Assurance

✅ **Compliance reporting established** - Metrics on consent, withdrawals, access logs, and incidents

✅ **Annual review scheduled** - Purpose, lawful basis, DPIA, and privacy notices reviewed annually or on change

✅ **Incident response readiness tested** - Breach scenarios involving location/IP data included in exercises

## Specific Attestations

### Location Tracking Compliance

We attest that:
- Location tracking requires explicit consent before processing
- Consent mechanism is implemented and functional
- Withdrawal mechanism is available and functional
- Location data is classified as PECR-regulated network-derived location data
- Value-added service justification is documented
- Security measures are in place for location data
- Retention policies are enforced
- Audit logging is implemented

### IP Address Tracking Compliance

We attest that:
- IP address collection is justified by legitimate interests (security) or consent (analytics)
- Cookie consent mechanism covers IP tracking for analytics
- IP addresses are anonymized after retention periods
- Security measures are in place for IP data
- Retention policies are enforced
- Audit logging is implemented
